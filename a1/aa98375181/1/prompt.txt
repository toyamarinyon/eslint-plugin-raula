Review feedback: good direction and tests mostly cover the intent. I found one issue to tighten before accepting.

`getArrayItemIndent()` currently does `source.slice(openIndex + 1)` and searches to the end of the file. For an empty config array like `export default defineConfig([]);` followed later by an unrelated indented block, it can incorrectly treat that later block's indentation as an existing array item indent. Please scope the existing-array-item lookup to the config array body only, using the already computed `closeIndex` from `findMatchingBracket`. If there is no item indent inside that array body, fall back to `lineIndent + detectIndentUnit(source)`.

Suggested shape:
- `getArrayItemIndent(source, openIndex, closeIndex, indentUnit)`
- Search only `source.slice(openIndex + 1, closeIndex)` for the first non-empty indented line.
- Add/adjust a test for an empty config array followed by an unrelated indented block, proving the fallback is file-level detection rather than scanning beyond the config array. Existing tests should still pass.
