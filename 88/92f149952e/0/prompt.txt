eslint-plugin-raulaを追加した後に、eslint.config.mjs を更新するのが手間なので、更新するスクリプトをnpx eslint-plugin-raula install` で実行できるようにしたいです。

1. プロジェクトのeslint configファイルを探す（eslint configのファイル形式でfizzyに検索）見つからなければwarnログ
2. 見つかったら変更内容をdiffでログ表示 Yes/No 確認
3. AGENTS.md の更新可否を確認 Yes/No (今のinstruct相当）
4. 実行

unjs/consola 使ってみやすく実装できるなら使ってもいいですが、小さく独自実装できるならそれでもいいです。
まずは作るものや作り方をディスカッションしましょう。

テストケースのconfig実際に `create-next` で作られるものと違うので、

```
import { defineConfig, globalIgnores } from "eslint/config";
import nextVitals from "eslint-config-next/core-web-vitals";
import nextTs from "eslint-config-next/typescript";

const eslintConfig = defineConfig([
  ...nextVitals,
  ...nextTs,
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    ".next/**",
    "out/**",
    "build/**",
    "next-env.d.ts",
  ]),
]);

export default eslintConfig;
```これで試しましょう。
